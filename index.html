The Present (Q1 2008)

The Problems

Redundant License Metadata

  • XML to drive API
  • RDF to serve to other consumers
  • Comphrensive list of licenses: which one?

Multiple Locations for "Engine" Information

  • XSLT rules
  • XML to drive "questions"
  • Jurisdiction information — both a question and a license property

Custom Formats, not so great

  • API clients become CC-specific
  • How do I format my "answers"?
  • How can I verify that I'm getting the correct answer?

Solutions

  • RDF as the canonical source of information
  • A single wrapper library for our services
  • SPARQL queries that can be used to query for specific types of information

CC in OOo

  • OpenOffice.org add-in to support CC licensing documents
  • Initial test-bed for pure-RDF/SPARQL approach

cc.license

  • Python library which provides a standard API for license information
  • All license metadata comes from RDF
  • Exploring replacing our "questions" definition with more RDF
  • Goal: Drop in ccREL metadata for any license (CC or not) and go
